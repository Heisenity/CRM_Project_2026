# Staff Feature Access

**Feature name:** Staff Feature Access  
**Route:** `/staff-feature-access`

## What this feature does
Staff Feature Access lets authorized users control which pages and tools each staff member can use.

You can use this page to:
- Select a staff member.
- Enable features they need for their role.
- Disable features they should not access.
- Save and apply permission updates.

## When users should use it
Use this page when you need to:
- Give a new staff member access to specific features.
- Remove access when responsibilities change.
- Restrict access for security or role separation.
- Fix cases where a staff user cannot open a required page.

## Step-by-step instructions
1. Open the app, then click **Staff Feature Access** in the left sidebar.
2. Use search (if available) to find the staff member.
3. Select the correct employee from the list.
4. Review the available feature toggles/check options.
5. Turn ON features the employee should use.
6. Turn OFF features the employee should not use.
7. Click **Save** (or equivalent action button) to apply changes.
8. Ask the employee to refresh, sign out/sign in, or reopen Staff Portal.
9. Confirm the employee can see only the allowed features.

## Expected result
Each staff user gets access only to the features required for their job.

Typical outcomes:
- Newly enabled features appear in the staff user’s portal.
- Removed features are no longer accessible.
- Access control stays aligned with role responsibilities.

## Common mistakes or errors
- Changes not visible: Ask user to refresh or sign in again.
- Wrong user updated: Verify employee name and ID before saving.
- User still has blocked access: Confirm required feature was enabled.
- User sees too many features: Reopen settings and disable extra permissions.
- Save failed: Retry after checking network and selected employee.
