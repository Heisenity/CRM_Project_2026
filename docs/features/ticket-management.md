# Support Ticket Management

**Feature name:** Support Ticket Management  
**Routes:** `/tickets`, `/tickets/new`

## What this feature does
Support Ticket Management helps teams record issues, track progress, and close requests in an organized way.

You can use this page to:
- Create new support tickets.
- View all tickets in one place.
- Filter tickets by status.
- Update ticket details and priority.
- Track ticket progress until closure.

## When users should use it
Use this page when you need to:
- Report a customer or internal issue.
- Follow up on open support work.
- Check which tickets are pending or resolved.
- Update ticket status after action is taken.

## Step-by-step instructions
1. Open the app, then go to **Tickets** in the left sidebar.
2. Review ticket tabs such as **All**, **Open**, **Resolved**, and **Closed**.
3. Use search and filters to find a specific ticket.
4. To create a ticket, click **Create Ticket** (or open `/tickets/new`).
5. Fill required details (reporter/customer, problem type, priority, and issue description).
6. Submit the ticket.
7. Open a ticket from the list to review full details.
8. Update status as work progresses (for example Open -> In Progress -> Resolved/Closed).
9. Save changes and confirm the ticket appears in the correct tab.

## Expected result
Issues are tracked from creation to closure with clear status and follow-up history.

Typical outcomes:
- New issues are logged quickly.
- Team members can see current ticket priority and status.
- Completed issues are moved to resolved/closed view.

## Common mistakes or errors
- Ticket not visible: Check active tab and filters.
- Cannot submit ticket: Complete required fields before submitting.
- Wrong priority selected: Edit ticket and save corrected priority.
- Status not updating: Refresh and verify save confirmation.
- Duplicate ticket created: Search existing tickets before creating a new one.
