# Login and Access

**Feature name:** Login and Access  
**Routes:** `/`, `/landing`, `/admin-login`

## What this feature does
Login and Access is the entry point to the application.

This page helps different users sign in and go to the correct workspace:
- Customer users go to Customer Portal.
- Staff users go to Staff Portal.
- Admin users go to Dashboard.

## When users should use it
Use this page when you need to:
- Sign in to the system.
- Sign in with a different account.
- Move to your role-based workspace.

## Step-by-step instructions
1. Open the app. You will see the login options on the landing page.
2. Choose the correct login section:
   - **Customer Login** for customer accounts.
   - **Staff Login** for employee/staff accounts.
   - Use **Admin Login** from the footer (**Admin Access**) for admin accounts.
3. Enter your login details carefully:
   - Customer: customer ID and phone.
   - Staff: employee ID and password.
   - Admin: admin ID, email, and password.
4. Click the login button.
5. Wait for automatic redirect.
6. Confirm you are on the correct page after login.

## Expected result
You are signed in successfully and redirected to your correct workspace.

Typical outcomes:
- Admin account opens **Dashboard**.
- Staff account opens **Staff Portal**.
- Customer account opens **Customer Portal**.

## Common mistakes or errors
- Wrong ID, email, phone, or password: Recheck each field and try again.
- Using the wrong login section: Switch to the correct user type before signing in.
- Password not visible: Use the show/hide password icon if available.
- Page keeps loading or does not redirect: Refresh once and sign in again.
- Redirect goes to unexpected page: Sign out and log in with the correct account type.
